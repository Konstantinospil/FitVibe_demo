## Executive Summary

This non-functional requirement defines internationalization standards and constraints for the FitVibe platform.

Support multiple languages and locales to serve international users.

## Business Context

- **Business Objective**: Support multiple languages and locales to serve international users.
- **Success Criteria**: English and German translations available, language preference persists, and locale formatting works.
- **Target Users**: All users (i18n affects everyone)

## Traceability

- **PRD Reference**: PRD §i18n
- **TDD Reference**: TDD §i18n

## Non-Functional Requirements

### Translation Support

The system shall support multiple languages:

- **Supported Languages**: English (EN) and German (DE)
- **Static Keys**: Static translation keys exist for all user-facing strings
- **Missing Key Detection**: Missing-key linter passes CI with 0 missing keys
- **No Hard-Coded Strings**: No hard-coded user-facing strings in covered paths

### Language Management

- **Language Toggle**: Users can toggle language preference
- **Persistence**: Language preference persists across sessions
- **Auto-Detection**: Language auto-detected from browser/profile
- **Fallback**: Fallback to English if translation missing

### Locale Formatting

- **Date Formatting**: Dates formatted according to locale
- **Number Formatting**: Numbers formatted according to locale
- **Currency Formatting**: Currency formatted according to locale (if applicable)

## Dependencies

### Technical Dependencies

- i18n library (i18next)
- Translation files (JSON)
- Locale formatting library

### Feature Dependencies

- All frontend features must use i18n

## Constraints

### Technical Constraints

- Static keys for EN/DE
- 0 missing translation keys
- No hard-coded strings

### Business Constraints

- English must always be available
- Translations must be accurate

## Assumptions

- Users understand language selection
- Translations are accurate
- Locale formatting is correct

## Risks & Issues

- **Risk**: Translations may be incomplete
- **Risk**: Locale formatting may be incorrect
- **Risk**: Missing keys may break functionality
