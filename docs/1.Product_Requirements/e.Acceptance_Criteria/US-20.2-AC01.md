## Criterion

Database volumes are encrypted at rest in production using filesystem-level encryption (e.g., encrypted EBS volumes, encrypted Kubernetes PersistentVolumes). Docker and Kubernetes configurations specify encrypted storage classes.

**SMART Criteria Checklist**:

- **Specific**: Database volumes encrypted at rest in production via filesystem encryption
- **Measurable**: Encryption verified via infrastructure configuration and storage class settings
- **Achievable**: Standard cloud provider or filesystem encryption
- **Relevant**: Data protection at rest
- **Time-bound**: Applied to all production deployments

## Test Method

Ops review verifies encrypted volumes are configured correctly. Security audit confirms encryption is enabled.

## Evidence Required

- Docker Compose configuration with encrypted volumes
- Kubernetes PersistentVolume configuration with encrypted storage class
- Cloud provider encryption settings (if applicable)
- Security audit report confirming encryption
