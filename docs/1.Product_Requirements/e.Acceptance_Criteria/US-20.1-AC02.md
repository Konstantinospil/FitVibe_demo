## Criterion

Development environments support relaxed SSL configuration (self-signed certificates with `rejectUnauthorized: false`) for local development while maintaining secure defaults for production.

**SMART Criteria Checklist**:

- **Specific**: Development environments allow relaxed SSL, production requires strict SSL
- **Measurable**: Environment-specific SSL configuration verified via `NODE_ENV` or `PGSSL` variable
- **Achievable**: Standard environment-based configuration pattern
- **Relevant**: Developer experience and security
- **Time-bound**: Applied in all environments

## Test Method

Integration tests verify SSL configuration works correctly in both development and production environments.

## Evidence Required

- Environment-specific SSL configuration code
- Test results for development and production SSL modes
- Documentation for local SSL setup
