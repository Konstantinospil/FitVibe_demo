## Overview

This directory contains formal workflow definitions for the FitVibe multi-agent system. These workflows define how agents coordinate to complete development tasks, from feature development to bug fixes and emergency hotfixes.

---

## Workflows

### 1. Feature Development Workflow
**File**: `feature-development.md`  
**Purpose**: Standard workflow for implementing new features  
**Status**: ✅ Active

### 2. Bug Fix Workflow
**File**: `bug-fix.md`  
**Purpose**: Systematic workflow for identifying, analyzing, and fixing bugs  
**Status**: ✅ Active

### 3. Emergency Hotfix Workflow
**File**: `emergency-hotfix.md`  
**Purpose**: Expedited workflow for critical production issues  
**Status**: ✅ Active

## Workflow Diagrams

All workflows include Mermaid diagrams for visual representation. See individual workflow files for details.

## Usage

Workflows are referenced by:
- **Planner Agent**: For orchestrating agent handoffs
- **Agent Executor**: For executing workflows programmatically
- **Documentation**: For understanding system behavior
